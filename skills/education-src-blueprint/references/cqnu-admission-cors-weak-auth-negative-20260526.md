# CQNU 招生系统 CORS + 弱鉴权负证据门禁（2026-05-26）

## 适用场景
教育 SRC 中发现招生/录取/通知书接口存在：
- 任意 `Origin` 回显；
- `Access-Control-Allow-Credentials: true`；
- 未登录可访问查询页、通知书下载页或模板打印页；
- 查询接口能进入业务逻辑但未命中真实数据。

## 结论
不要仅凭 “CORS 任意 Origin + Credentials=true + 招生页面可访问” 单独提交。只有补到真实敏感数据读取、跨用户对象读取、通知书真实内容或已登录用户被恶意 Origin 跨域读到敏感响应，才建议提交。

## 本次验证要点
目标：`zsxt.cqnu.edu.cn` 重庆师范大学招生管理系统。

已确认：
1. 多个接口会回显恶意 Origin，并允许 Credentials。
2. 录取查询入口无需登录访问，页面字段包括姓名、考生号、身份证号、准考证号、EMS 单号。
3. 招生官网公开拟录取名单页面存在，含姓名、脱敏身份证号、录取专业。
4. 随机不存在路径返回 302 空正文，说明招生页面不是简单 SPA fallback。

未确认：
1. 测试姓名/证件号查询只返回“很抱歉，没有你的录取信息”。
2. 用公开名单中的姓名 + 脱敏身份证格式查询仍不命中真实录取信息。
3. 通知书下载接口对空值或 `00000000000000` 等无效 `ksh` 返回的是壳页面，隐藏字段为空。
4. 通知书模板打印接口返回的是通用模板字段（姓名、录取编号、条形码等占位符），不是某个真实考生数据。

## 低影响验证流程
1. 先验证 CORS：
   `curl -k -i -H 'Origin: https://evil.example' 'https://TARGET/path'`
2. 找随机不存在路径作为对照，排除 SPA fallback / 统一错误：
   `curl -k -i -H 'Origin: https://evil.example' 'https://TARGET/nonexistent_hermes_control_<ts>'`
3. 如果招生官网有公开拟录取名单，只能作为低影响样本来源；名单里通常是脱敏身份证，不能视为未授权读取证据。
4. 用公开脱敏样本请求查询接口后，若仍只返回“没有录取信息”，应归为负证据。
5. 对通知书下载/模板接口必须检查是否出现真实姓名、专业、EMS、通知书编号、条形码值等实例数据；只有模板占位字段不算泄露。
6. 记录完整 request/response/header、body hash、随机路径对照和最终判断，不要输出可提交报告。

## 提交门槛
满足以下任一项才建议报告：
- 未授权读取真实录取结果、EMS 单号、通知书内容或完整/半完整 PII；
- `ksh` / `mbid` 可预测或可枚举，并返回真实考生通知书；
- 存在无需登录返回考生列表、录取批次明细、通知书数据的 API；
- 已登录考生或招生人员访问恶意页面时，浏览器可跨域读取到敏感响应。

## 不建议提交的证据形态
- 只有 CORS 任意 Origin 回显；
- 只有招生查询页面公开可访问；
- 只有“进入业务逻辑但返回没有你的录取信息”；
- 只有公开官网名单中的脱敏姓名/身份证；
- 只有通知书模板字段，没有真实数据；
- 只有壳页面、空隐藏字段或通用页面。

## 报告处理建议
若平台或用户仍要求保留材料，应写成“最终决策/补充证据版”，标题明确“不建议单独提交，建议作为补充证据”，并把未证明的点写清楚，避免夸大为考生数据泄露。
