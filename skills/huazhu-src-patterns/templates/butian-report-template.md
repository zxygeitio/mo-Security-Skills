## 补天等级判定

### 严重
- RCE/命令执行/上传webshell
- 核心DB SQL注入+大量用户数据泄露
- 前端JS泄露AppSecret→未授权访问用户数据

### 高危
- SQL注入(获取user)
- CORS反射型+Credentials=true(认证接口)
- 未授权访问管理后台
- API未授权+IDOR遍历
- 越权访问敏感数据

### 中危
- 普通信息泄露(源码/日志/配置)
- Java堆栈信息泄露
- 测试环境公网暴露
- CORS通配符*+Credentials=true

### 低危
- 版本号/phpinfo泄露
- 反射型XSS
- URL跳转
- 短信轰炸
- 用户IP未授权获取

## 类型映射
| 发现 | 填写类型 |
|------|---------|
| CORS反射 | 安全配置错误 |
| .env/.git泄露 | 信息泄露 |
| API无认证 | 未授权访问 |
| IDOR遍历 | 越权访问 |
| SQL注入 | SQL注入 |
| 短信轰炸 | 安全机制绕过 |
