## 关键漏洞发现

### 1. ezm.ys7.com Next.js __NEXT_DATA__ 信息泄露
Next.js SSG页面在 `<script id="__NEXT_DATA__">` 中泄露:
- 内部Windows开发路径: `F:\\workspace\\rtcwork\\rtc_portal\\branches\\v3.1.5\\public\\locales`
- 调试模式: `"debug":true`
- 客服手机: 13148352979
- 百度统计ID: d216aa98b35a7747739058994ebb603

**检测方法**: `curl -sk https://ezm.ys7.com/ | grep -o '__NEXT_DATA__[^<]*'`

### 2. store-front.ys7.com config.js 配置泄露
Vue.js SPA的 `/config.js` 暴露:
- ecadmin.ys7.com (管理后台)
- vending-gateway.ys7.com:8586 (WebSocket)
- mall-gateway.ys7.com / mall-webgateway.ys7.com
- fs.ys7.com/upload.php (文件上传)
- Geetest CAPTCHA ID: f9350603a2daf430a742bd784316546f
- appId: 6126e42c790f49ef8c7a
- Baidu Maps API key: cwo0RxtBi9KYHdC2bIRSkMa0CiYmrVMp

**Apollo配置端点**: `/mall_newstore_front-nodeApolloCachedConfig.json`

### 3. ifttt.ezvizlife.com OAuth2 + 堆栈泄露
- 框架: Spring Security + Alibaba Druid + SiteMesh + Tengine
- 内部包: `com.hikvision.oauth2.web.filter.CustomSecurityContextPersistenceFilter`
- /login.jsp → 500 NullPointerException (完整堆栈)
- /druid/ → 403 (存在但被nginx拦截)
- /oauth/authorize 接受任意client_id, 存入CLIENTID cookie
- SAVED_REQUEST cookie 存储OAuth请求

### 4. openapicn.eziot.com Swagger UI
- /swagger-ui.html → 200 (Springfox 2.8.0-SNAPSHOT)
- /api/lapp/token/get → 200 (IoT token端点)
- /api/lapp/device/list → 200 (设备列表端点)
- appKey格式: 以"at"开头 (如at9d3d4sad)
- 错误码: 10030=appKey存在但secret不匹配, 10017=appKey不存在

### 5. vending-gateway.ys7.com:8586 版本泄露
- workerman/4.0.37 (PHP WebSocket框架)
- 响应: `<h1>WebSocket</h1><hr>workerman/4.0.37</h1>`

## IoT API appKey枚举发现
- appKey前缀: "at" + 8位字母数字
- 常见appKey格式: at9d3d4sad, at79fqsszs, at9rfsax3s
- 区分错误码可判断appKey是否存在: 10030=存在(secret错), 10017=不存在

## 区域化认证服务器
auth.sq.ys7.com 和 auth.xy.ys7.com 是区域认证服务器:
- X-Powered-By: PS (可能为内部框架标识)
- /doLogin → 200 (返回retcode 1007=密码错误)
- /captcha → 200 (JPEG验证码)
- /druid → 302 (Druid存在但被拦截)
- /login → 302
