# CTF Writeup Format

User-preferred writeup structure for competition submissions:

```
题目名称：XXX
题目类型：WEB/Crypto/Misc/PWN/Reverse
难度：初级/中级/高级
分值：XXX分

一、解题过程

1. 获取到题目附件/源码，分析发现XXX漏洞/特征。
2. 利用XXX工具/技术进行XXX操作。
3. 利用XXX编码/加密方式构造Payload/解密数据。
4. 获取到flag：flag{xxx}

二、漏洞分析

简要说明漏洞成因和利用原理（2-3句话）。
```

## Style Notes
- 简洁不啰嗦，但关键步骤要清晰
- 不用代码块包裹整篇writeup，直接输出对话
- 步骤编号 1-4 对应：发现→工具→利用→flag
- 分析部分点到为止，不长篇大论
