## IoT开放平台API (open.ys7.com / openapicn.eziot.com)

### API认证流程
```
1. 注册开发者账号 → 获取appKey+appSecret
2. POST /api/lapp/token/get {appKey, appSecret} → 获取accessToken
3. 使用accessToken调用设备API
```

### 关键API端点
```
POST /api/lapp/token/get → 获取token (需appKey+appSecret)
POST /api/lapp/token/refresh → 刷新token
POST /api/lapp/device/list → 设备列表
POST /api/lapp/device/info → 设备详情 (需deviceSerial)
POST /api/lapp/camera/list → 摄像头列表
POST /api/lapp/video/live/playAddress → 实时视频流
POST /api/lapp/video/history/playAddress → 历史视频
POST /api/lapp/alarm/list → 告警列表
POST /api/lapp/device/capture → 设备截图
POST /api/lapp/config/version → 配置版本
POST /api/lapp/config/get → 获取配置
```

### API错误码
```
10001 = 参数为空
10002 = accessToken过期
10017 = appKey不存在 (open.ys7.com)
10030 = appkey和appsecret不匹配 (openapicn.eziot.com)
-1 = 未知错误
```

### appKey格式
- 前缀: at + 8位小写字母数字
- 示例: at8ksszsss, at9d3d4sad, at79fqsszs, at9rfsax3s

## ThinkPHP RCE WAF防护 (store.ys7.com)

### 三重WAF
- openRASP: 拦截phpinfo/system/assert等(返回403)
- 阿里云WAF: 拦截命令注入(返回418:6357, 连接杀死000:0)
- Cloudflare: store-front等应用

### 已尝试绕过(全部失败)
- ThinkPHP invokefunction RCE → 空响应
- POST _method=__construct → 418
- Unicode/双重编码 → WAF拦截
- HTTP走私(CL.TE/TE.CL) → Tengine 400
- X-Forwarded-Host绕过 → 000:0
- passthru/popen差异 → 仍被拦截

## OAuth认证系统 (openauth.ys7.com)

### OAuth流程
```
1. GET /oauth/authorize?client_id=X&redirect_uri=Y&response_type=code
2. 用户登录 → 获取authorization_code
3. POST /oauth/token {grant_type=authorization_code, code, redirect_uri}
```

### 开放重定向 (需用户交互)
- redirect_uri参数未做白名单校验
- JS代码: params += '&' + location.search.substring(1);
- 攻击链: 钓鱼→用户登录→授权码泄露
